Gerência de Configuração e Controle de Versão

Controle de Versão

  • Os sistemas de controle de versão são programas de software que ajudam a controlar as alterações feitas no código ao longo do tempo. Conforme o desenvolvedor edita o código, o sistema de controle de versão tira um instantâneo dos arquivos. Em seguida, ele salva esse instantâneo permanentemente para que ele possa ser recuperado mais tarde, se necessário.
    • Fonte - https://learn.microsoft.com/pt-br/devops/develop/git/what-is-version-control

Controle de Mudanças

  • O processo de controle de mudanças é uma forma de os gestores de projeto enviarem solicitações para análise pelos participantes, que, devem então aprovar ou rejeitar as sugestões. Trata-se de um processo importante para ajudar na gestão de grandes projetos com vários componentes dinâmicos.
    • Fonte - https://asana.com/pt/resources/change-control-process

Integração Contínua

  • A definição de CI é a seguinte: é a prática de fazer com que todos os que estão trabalhando no mesmo projeto mesclem regularmente suas alterações na base de código em um repositório central. Como os projetos geralmente têm mais de um desenvolvedor envolvido, é importante que as peças do trabalho sejam combinadas num local central. O ideal é que esse processo seja automatizado e realizado várias vezes ao dia. O objetivo da integração contínua é manter uma abordagem confiável para a construção e lançamento de um software, promovendo colaboração, automação e ciclos curtos de feedback.
    • Fonte - https://www.dio.me/articles/o-que-e-integracao-continua-ci
